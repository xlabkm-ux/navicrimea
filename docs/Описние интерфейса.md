# Описние интерфейса

## 1. Цель
Снизить визуальную перегрузку галереи и перевести главный интерфейс в формат списка предложений с быстрым сравнением и выбором.

## 2. Главный сценарий пользователя
1. Турист выбирает категорию предложения.
2. Турист сортирует список по критерию.
3. Турист просматривает краткие карточки с ключевой пользой.
4. По нажатию открывает полную карточку объекта.

## 3. Структура экрана списка
- Блок категорий: горизонтальные кнопки (как на карте).
- Блок сортировки: выпадающий список критериев.
- Блок фильтров: семейный отдых, питомцы, бесплатная отмена.
- Лента карточек предложений.

## 4. Карточка предложения (краткая)
- Фото-превью.
- Название объекта/предложения.
- Локация (район).
- Цена "от".
- Рейтинг и число отзывов.
- Дистанция до моря.
- Бейджи ценности: скидка, проверено, срочное предложение.

## 5. Полная карточка объекта
- Открывается из элемента списка.
- Содержит заглушку полной структуры:
  - Полное описание.
  - Удобства.
  - Условия/правила.
  - Включено в услугу.
- Контент полной карточки уточняется на следующем этапе.

## 6. Интерфейс обсуждения и доработки
- Отдельный экран "Проектирование".
- Показ документов проектирования прямо в интерфейсе.
- Форма комментариев по доработке.
- Статусы комментариев: новый, в работе, запланирован, готово.

## 7. Критерии сортировки
- Рекомендуемое.
- Цена по возрастанию.
- Цена по убыванию.
- Ближе к морю.
- По рейтингу.
- По скидке.
- Новые предложения.

## 8. KPI для проектирования
- Сокращение времени выбора предложения.
- Рост кликов в полную карточку.
- Рост конверсии в бронирование/заявку.
- Снижение отказов на главном экране.

## 9. Референс-паттерн
- Экран регионов и отелей: https://travel.yandex.ru/hotels/
- Экран списка предложений: https://travel.yandex.ru/hotels/zelenogradsk/

Перенимаемые элементы:
- Левый блок фильтров с быстрыми опциями.
- Сортировка результатов в шапке списка.
- Карточка предложения с ценой, скидкой, рейтингом, расстоянием до моря.
- Быстрый CTA "Выбрать".
- Блок быстрого сравнения 2-3 объектов.
