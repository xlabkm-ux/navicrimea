# База данных, структура и данные

## 1. Назначение
База данных поддерживает:
- Категории предложений.
- Список карточек предложений для интерфейса.
- Данные полной карточки объекта.
- Комментарии по обсуждению и доработке.

## 2. Таблицы

### 2.1 listing_categories
- `id` TEXT PRIMARY KEY
- `title` TEXT NOT NULL
- `icon` TEXT
- `sort_order` INTEGER DEFAULT 0

Назначение: справочник категорий интерфейса списка.

### 2.2 listing_offers
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `object_id` INTEGER NULL (связь с таблицей `objects`)
- `category_id` TEXT NOT NULL (FK -> `listing_categories.id`)
- `title` TEXT NOT NULL
- `short_description` TEXT
- `region` TEXT
- `district` TEXT
- `distance_to_sea_m` INTEGER
- `distance_to_center_m` INTEGER
- `price_from` INTEGER
- `rating` REAL DEFAULT 0
- `reviews_count` INTEGER DEFAULT 0
- `has_discount` INTEGER DEFAULT 0
- `discount_percent` INTEGER DEFAULT 0
- `is_verified` INTEGER DEFAULT 0
- `free_cancellation` INTEGER DEFAULT 0
- `breakfast_included` INTEGER DEFAULT 0
- `pets_allowed` INTEGER DEFAULT 0
- `family_friendly` INTEGER DEFAULT 0
- `urgent_badge` TEXT
- `cover_image_url` TEXT
- `created_at` TEXT DEFAULT datetime('now')
- `updated_at` TEXT DEFAULT datetime('now')

Назначение: основной список карточек предложений.

### 2.3 listing_offer_details
- `offer_id` INTEGER PRIMARY KEY (FK -> `listing_offers.id`)
- `full_description` TEXT NOT NULL
- `amenities_json` TEXT
- `policies_json` TEXT
- `included_json` TEXT
- `updated_at` TEXT DEFAULT datetime('now')

Назначение: хранение данных полной карточки объекта.

### 2.4 design_feedback
- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `topic` TEXT NOT NULL
- `comment` TEXT NOT NULL
- `author` TEXT DEFAULT 'Команда'
- `status` TEXT CHECK in (`new`, `in_review`, `planned`, `done`)
- `priority` TEXT CHECK in (`low`, `normal`, `high`)
- `created_at` TEXT DEFAULT datetime('now')
- `updated_at` TEXT DEFAULT datetime('now')

Назначение: обсуждение и управление доработками.

## 3. Индексы
- `idx_listing_offers_category` по `listing_offers(category_id)`
- `idx_listing_offers_sort` по `listing_offers(price_from, rating, distance_to_sea_m)`

## 4. Начальные данные (seed)
- Категории: отели, апартаменты, гостевые дома, еда, активности.
- Предложения:
  - Автогенерация из таблицы `objects`.
  - Дополнительные записи для категорий `food` и `activities`.
- Таблица `design_feedback` содержит стартовую задачу обсуждения.

## 5. API для интерфейса
- `GET /api/v2/listings/categories`
- `GET /api/v2/listings/offers`
- `GET /api/v2/listings/offers/:id`
- `GET /api/v2/design/docs`
- `GET /api/v2/design/feedback`
- `POST /api/v2/design/feedback`
- `PATCH /api/v2/design/feedback/:id`
